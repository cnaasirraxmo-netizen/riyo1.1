# RIYOBOX - Full Integration & Setup Guide

This guide provides the complete blueprint for making RIYOBOX "real working" across Backend, Mobile, and Web.

## 1. Unified Authentication (NEW)

The backend now supports **Unified Authentication**. This means you can log in via Firebase (Google, Email/Password) OR via our internal JWT system. The backend middleware automatically detects and verifies the token type.

### Token Support
- **Firebase ID Token**: Sent by Web (React) and Mobile (Flutter) after Firebase login.
- **Internal JWT**: Generated by `/auth/register` and `/auth/login` for non-Firebase users.

---

## 2. Backend Setup (Node.js)

### Environment Variables (.env)
```env
MONGO_URI=your_mongodb_uri
JWT_SECRET=your_jwt_secret

# Firebase Configuration (Choose one)
# Option A: Full JSON string (Recommended for Render.com/Heroku)
FIREBASE_SERVICE_ACCOUNT_JSON={"type": "service_account", "project_id": "...", ...}

# Option B: Path to JSON file
FIREBASE_SERVICE_ACCOUNT_PATH=config/firebase-service-account.json

# Sports API
FOOTBALL_API_KEY=your_api_football_key
```

### Key Fixes Implemented
- **E11000 Duplicate Key**: Resolved by using `sparse: true` on `firebaseUid`.
- **OverwriteModelError**: Resolved by using safe model compilation patterns in Mongoose.

---

## 3. Flutter App Setup (Mobile)

### Avoiding "Firebase not found" / White Screen
1. **Google Services**: Ensure `android/app/google-services.json` is present.
2. **Resilience**: The app now has a 5-second timeout for Firebase initialization. If it fails, it starts in "Mock Mode" rather than hanging.
3. **Provider**: `AuthProvider` now checks `isFirebaseAvailable` before calling Firebase methods.

### Debugging
If you see "Firebase initialization failed" in logs:
- Check if your `google-services.json` matches your package name `com.example.riyobox`.
- Run `flutter clean` then `flutter pub get`.

---

## 4. Web Platform Setup (React)

### Config Initialization
Update `web_user/src/utils/firebase.js` and `web_admin/src/utils/firebase.js` with your Firebase project config object:
```javascript
const firebaseConfig = {
  apiKey: "...",
  authDomain: "...",
  projectId: "...",
  storageBucket: "...",
  messagingSenderId: "...",
  appId: "..."
};
```

### Authentication Flow
The web apps are configured to log in via Firebase first, then sync with the backend by sending the Firebase ID Token in the `Authorization: Bearer <token>` header.

---

## 5. Deployment Checklist

1. **MongoDB**: Ensure your IP is whitelisted (or 0.0.0.0/0 for production).
2. **Firebase**: Enable **Email/Password** and **Google** providers in the Firebase Console.
3. **Admin Access**: To make yourself an admin, manually update your user record in MongoDB: `db.users.updateOne({email: "your@email.com"}, {$set: {role: "admin"}});`
